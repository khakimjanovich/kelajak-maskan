# Kelajak-Maskan README Marketing Design

**Date:** 2026-03-24

**Audience:** Potential adopters evaluating the product

**Positioning:** Sell the current reality first, then point to the next product step without overstating what already exists.

## Brand Direction

Kelajak-Maskan should read as an aspirational, brand-led product for disciplined builders and developers. The name should carry the idea of "a place for the future" rather than a generic software repository.

## Core Message

The README should frame the product around discipline, continuity, and grounded iteration:

- builders lose momentum between sessions
- intent, constraints, and outcomes get scattered
- Kelajak-Maskan gives that discipline a durable home

## Current Reality To Highlight

The current repo already provides:

- local SQLite-backed project history
- domain records for wants, validations, plan revisions, actions, outcomes, and audit logs
- CLI access to latest want, open cycle, and summary views

## Promised Next Step

The next version should be described as a more accessible product surface built on the current history engine. The README should avoid claiming a shipped UI or API today.

## README Structure

1. Product title and value proposition
2. Why Kelajak-Maskan exists
3. What it does today
4. Who it is for
5. What's coming next
6. Getting started
7. Current status

## Copy Constraints

- Remove all stock Laravel marketing copy, badges, and framework sections
- Keep implementation details secondary to product value
- Stay honest about the current CLI-first state
- Keep the README updated per iteration to reflect shipped functionality first
